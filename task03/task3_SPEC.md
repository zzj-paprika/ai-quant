# 博杰股份双均线 + RSI 风控回测实验 — 产品与算法规格

## 1. 项目目标

以博杰股份（`002975.SZ`）为示例，制作一个可复现的 Jupyter Notebook 和一个可交互的网页应用，用于测试“双均线交叉 + 可选 RSI 过滤 + 自定义止盈止损”策略。

用户必须能在网页中自由调整 `SMA(x, y)` 的快、慢周期；Notebook 则按“数据检查 → 指标计算 → 信号生成 → 逐日回测 → 绩效评估 → 结果解释”的顺序展示完整算法过程。

本项目用于量化研究教学，不构成投资建议，也不把历史回测结果解释为未来收益承诺。

## 4. 策略的唯一计算口径

### 4.1 参数

| 参数 | 符号/字段 | 默认值 | P0 范围 | 说明 |
|---|---|---:|---:|---|
| 快均线周期 | `x / fast_period` | 5 | `2 <= x < y` | 用户可自由输入整数，不限于预设组合 |
| 慢均线周期 | `y / slow_period` | 20 | `x < y <= N-2` | 用户可自由输入整数；`N` 为当前股票有效交易日数 |
| RSI 过滤开关 | `rsi_filter_enabled` | 开启 | 开启/关闭 | 关闭后金叉不再检查 RSI |
| RSI 周期 | `rsi_period` | 14 | 2–50 | 用户可输入整数，使用 Wilder RSI |
| RSI 入场上限 | `rsi_entry_max` | 70 | 1–99 | 入场要求严格 `< 阈值` |
| 止盈开关 | `take_profit_enabled` | 开启 | 开启/关闭 | 关闭后不检查止盈触价 |
| 止盈率 | `take_profit` | 10% | 0.1%–100% | 用户输入正百分比，相对实际买入成交价 |
| 止损开关 | `stop_loss_enabled` | 开启 | 开启/关闭 | 关闭后不检查止损触价 |
| 止损率 | `stop_loss` | 5% | 0.1%–99% | 用户输入正百分比，相对实际买入成交价 |
| 反向死叉退出 | `reverse_cross_enabled` | 开启 | 开启/关闭 | 关闭后死叉不产生卖出订单 |
| 初始资金 | `initial_cash` | ¥100,000 | ¥1,000–¥1,000,000,000 | 用户输入正数 |
| 万二手续费开关 | `commission_enabled` | 开启 | 开启/关闭 | 网页可切换，切换后立即重算 |
| 交易费率 | `commission_rate` | 0.02%/边 | 固定 | 开启时买卖各按成交额万分之二，关闭时为 0 |
| 最低佣金 | `min_commission` | ¥0 | 固定 | 简化模型 |
| 印花税/滑点 | — | 0 | 固定 | P0 不模拟，必须披露 |
| 回测起始日期 | `start_date` | 数据最早可用日 | 用户选择 | 含起始日；非交易日顺延至下一交易日 |
| 回测结束日期 | `end_date` | 数据最后可用日 | 用户选择 | 含结束日；非交易日回退至上一交易日 |

网页主界面要求自由输入 `x` 与 `y`、回测起止日期、RSI 参数、止盈止损和初始资金，并提供万二手续费及各规则开关。不得只提供若干固定参数组合。`N-2` 上限保证至少存在“形成交叉 → 下一交易日成交”的可能；这里 `N` 指截至所选结束日期可用于指标计算的交易日数。若只能显示均线但不能完成回测，界面应说明原因而不是生成误导性零交易结果。

### 4.2 回测日期与指标预热

- 用户输入的是日历日期，系统必须同时显示实际采用的首、末交易日。
- `start_date` 当天开盘前始终为空仓、现金为初始资金；不得把起始日前的持仓或待成交订单带入回测。
- SMA 和 RSI 可以读取起始日前的历史数据作为预热数据，但预热期不产生交易、不计入净值、不计入收益或绩效。
- 起始日收盘可以基于预热数据判断交叉，最早在下一交易日开盘成交。
- `end_date` 收盘后结束回测；当天收盘产生但需下一日执行的信号记为“未成交信号”。期末已有持仓按结束日收盘价估值，不强制卖出。
- 若所选起始日期早于数据最早日或结束日期晚于数据最后日，输入框可以保留用户值，但结果必须明确显示裁剪后的实际交易区间。
- 所有指标卡、交易记录、月度收益和图表主视窗只统计实际回测区间；价格/均线图可选展示预热区，但必须用浅色背景明确标成“不计入回测”。

### 4.3 SMA

对收盘价计算：

$$
SMA_n(t)=\frac{1}{n}\sum_{i=0}^{n-1}Close_{t-i}
$$

在窗口不足时返回 `NaN`，不得用不足 `n` 天的均值代替。

### 4.4 RSI(14)

RSI 使用 Wilder 平滑法，而非简单滚动平均：

```text
delta_t = close_t - close_(t-1)
gain_t  = max(delta_t, 0)
loss_t  = max(-delta_t, 0)

首个 avg_gain = 前 14 个有效 gain 的算术平均
首个 avg_loss = 前 14 个有效 loss 的算术平均
后续 avg_gain_t = (avg_gain_(t-1) × 13 + gain_t) / 14
后续 avg_loss_t = (avg_loss_(t-1) × 13 + loss_t) / 14
RSI_t = 100 - 100 / (1 + avg_gain_t / avg_loss_t)
```

边界定义：平均亏损为 0 且平均上涨大于 0 时 RSI=100；两者同时为 0 时 RSI=50。初始化窗口内 RSI 为 `NaN`。

### 4.5 入场信号

交易日 `t` 收盘后产生金叉信号，当且仅当：

```text
SMA_x(t-1) <= SMA_y(t-1)
AND SMA_x(t) > SMA_y(t)
AND（RSI 过滤关闭 OR RSI_period(t) < rsi_entry_max）
AND 当前为空仓
```

处理规则：

- 交叉使用“前一日小于等于、当日严格大于”，使均线相等后的首次突破只产生一次信号。
- RSI 过滤开启时条件严格为 `< rsi_entry_max`；RSI 等于阈值或无有效值时不入场。过滤关闭时，RSI 不参与入场判断，但仍可在图上展示。
- 信号只使用 `t` 日及以前数据，计划在下一交易日 `t+1` 开盘买入。
- 若 `t` 已是数据最后一日，记录为“未成交信号”，不创建持仓。

### 4.6 出场规则

持仓后同时启用以下三类退出：

1. **止盈**：开关开启且价格达到买入成交价的 `+take_profit`。
2. **止损**：开关开启且价格达到买入成交价的 `-stop_loss`。
3. **反向信号**：开关开启，且 `SMA_x(t-1) >= SMA_y(t-1)`、`SMA_x(t) < SMA_y(t)`，于下一交易日开盘卖出。

止盈价和止损价在买入成交时固定：

```text
take_profit_price = entry_price × (1 + take_profit)
stop_loss_price   = entry_price × (1 - stop_loss)
```

不使用扣费后的成本反推触发线；净收益率另行扣除买卖佣金。某规则关闭时，不计算也不触发对应价格线。允许只保留一种退出规则；若三个退出规则全部关闭，界面必须强提醒“仓位只能在期末估值”，但允许用户用于研究持有行为。

### 4.7 成交时序与同日冲突

为避免未来函数，每个交易日按以下顺序处理：

1. **开盘阶段**：执行前一交易日收盘后排队的买入或反向信号卖出。
2. **日内阶段**：仅对开盘后仍持有的仓位检查当天 OHLC 是否触发止损/止盈。
3. **收盘阶段**：用当天收盘价计算 SMA、RSI 与交叉，生成下一交易日订单。
4. **估值阶段**：按当天收盘价计算净值；若当天已经卖出，则使用卖出后的现金。

日内触价的成交价定义：

| 场景 | 成交价 |
|---|---|
| `open <= stop_loss_price` | 当日 `open`，表示向下跳空可能造成超额亏损 |
| `low <= stop_loss_price < open` | `stop_loss_price` |
| `open >= take_profit_price` | 当日 `open`，保留向上跳空收益 |
| `high >= take_profit_price > open` | `take_profit_price` |

若同一天 `low <= stop_loss_price` 且 `high >= take_profit_price`，日线数据无法判断先后，P0 采用保守假设：**止损优先**。交易记录增加 `ambiguous_intraday=True`，网页显示“同日双触发，按止损优先模拟”。

开盘执行反向卖出后，当天不再检查该旧仓位的止盈止损。开盘买入的新仓位允许从当日开盘之后检查止盈止损；若当天高低价同时覆盖两条线，同样按止损优先。

### 4.8 仓位、费用和期末处理

- 单标的、只做多、最多一笔持仓，不加仓、不做空。
- 每次买入使用全部可用现金，允许小数股，以突出策略逻辑并保持 Python/JavaScript 一致。
- `commission_enabled=True` 时，买入和卖出分别按成交额的 `0.0002`（万分之二）计费；`False` 时有效费率为 0。
- 买入佣金：`gross_buy × commission_rate`；股数应解方程 `shares = cash / (entry_price × (1 + commission_rate))`。
- 卖出净收入：`shares × exit_price × (1 - commission_rate)`。
- 所有收益和净值均包含上述佣金。
- 数据结束时不强制平仓；未平仓仓位按最后收盘价计入期末净值，并单独标为“浮动持仓”。胜率、平均持仓期等只统计已平仓交易。
- 同一交易日完成卖出后不允许再次入场，避免依赖无法由日线确认的盘中顺序。

## 5. 回测引擎输出契约

核心函数建议保持纯函数形式：

```python
result = run_backtest(
    bars,
    fast_period=5,
    slow_period=20,
    rsi_filter_enabled=True,
    rsi_period=14,
    rsi_entry_max=70,
    take_profit_enabled=True,
    take_profit=0.10,
    stop_loss_enabled=True,
    stop_loss=0.05,
    reverse_cross_enabled=True,
    initial_cash=100_000,
    commission_enabled=True,
    commission_rate=0.0002,
    start_date=None,
    end_date=None,
)
```

返回对象至少包含：

- `daily`: 日期、OHLC、SMA 快慢线、RSI、金叉/死叉、持仓、现金、持仓市值、策略净值、基准净值、回撤。
- `trades`: 每笔已平仓交易的进出日期、信号日期、成交价、股数、费用、毛/净收益、收益率、持仓交易日数、退出原因和日内冲突标记。
- `open_position`: 期末未平仓信息或 `null`。
- `pending_signals`: 数据末日无法成交的信号。
- `metrics`: 绩效指标。
- `metadata`: 参数、标的、完整数据范围、用户请求日期、实际回测交易日范围、预热行数、数据行数、复权口径和计算版本。

退出原因枚举固定为：

```text
take_profit | stop_loss | reverse_cross
```

## 6. 绩效指标

| 指标 | 口径 |
|---|---|
| 总收益率 | `ending_equity / initial_cash - 1` |
| 年化收益率 | `(ending_equity / initial_cash) ** (252 / effective_days) - 1` |
| 买入持有收益率 | 首个可交易日开盘买入、末日收盘估值，并使用相同双边费率；保证公平对比 |
| 超额收益 | 策略总收益率减买入持有收益率 |
| 最大回撤 | `min(equity / equity.cummax() - 1)` |
| 年化波动率 | 日净值收益标准差 `× sqrt(252)` |
| 夏普比率 | 日均收益 / 日收益标准差 `× sqrt(252)`，无风险利率设为 0 |
| 已平仓交易数 | 完整买卖回合数 |
| 胜率 | 净收益大于 0 的已平仓交易数 / 已平仓交易数 |
| 盈亏比 | 平均盈利金额 / 平均亏损金额绝对值；分母不存在时显示 `N/A` |
| 平均持仓期 | 已平仓交易的平均持仓交易日数 |
| 退出原因分布 | 止盈、止损、反向信号各自次数与占比 |

`effective_days` 使用净值序列相邻交易日区间数量。交易数为 0、波动率为 0 或样本不足时显示 `N/A`，不得输出无穷大或误导性的 0。

### 6.1 量化指标介绍文案

网页和 Notebook 必须为以下七项指标提供简短介绍。网页可使用指标名旁的信息按钮或可展开的“指标说明”，但含义不能只藏在鼠标悬停中：

| 指标 | 用户可见说明 |
|---|---|
| 总收益率 | 策略从回测开始到结束的总回报百分比；正值表示盈利，负值表示亏损。 |
| 年化收益率 | 将区间总收益按 252 个交易日折算为年化值，便于不同周期横向比较；不足一年时必须标注“短样本年化”。 |
| 胜率 | 盈利的已平仓交易次数 ÷ 已平仓交易总次数，表示历史交易中盈利的频率；高胜率不等于高收益。 |
| 盈亏比 | 平均单笔盈利 ÷ 平均单笔亏损绝对值，表示平均赚取金额相对平均亏损金额的倍数；大于 2 常被视为较有利，但必须结合交易次数、胜率和样本长度判断。 |
| 最大回撤 | 策略净值从历史高点到随后低点的最大跌幅，用于衡量样本内最严重的资金回落。 |
| 夏普比率 | 单位波动风险对应的历史超额收益，本项目设无风险利率为 0；大于 1 常被视为较好、大于 2 常被视为优秀，但短样本下可能非常不稳定。 |
| 超额收益 | 策略总收益率 − 买入持有总收益率，用于判断策略在相同区间和费用口径下是否跑赢买入持有基准。 |

介绍文案不得把阈值写成保证或评级结论。指标为 `N/A` 时，说明中展示原因，例如“没有已平仓亏损交易，盈亏比无法计算”。

### 8.3 图表与表格

P0 图表：

1. **策略净值与买入持有净值对比图**：两条曲线均以初始资金为起点，并使用相同的万二手续费开关口径。X 轴为交易日期，Y 轴可在“净值金额/归一化净值”间切换，默认显示净值金额。悬停展示日期、策略净值、基准净值和两者累计收益率；图例明确区分策略与基准。
2. **价格、均线与买卖信号图**：主图展示前复权（收益口径）收盘价、`SMA(x)` 和 `SMA(y)`，并叠加实际成交点，而不只是原始交叉信号。买入使用向上三角，卖出使用向下三角；止盈、止损、反向死叉使用不同形状或文字标签。悬停展示日期、价格、两条均线、RSI、信号日/成交日和退出原因。下方保留时间轴对齐的自定义周期 RSI 子图及用户设置的阈值线；RSI 过滤关闭时标注“仅展示，不参与信号”。
3. **策略回撤变化图**：按 `drawdown_t = equity_t / running_max_equity_t - 1` 绘制不高于 0 的面积图。标注最大回撤百分比、净值峰值日、谷底日；若样本结束时尚未恢复，可注明“尚未修复”。悬停展示当日回撤和此前净值高点。
4. **月度收益率统计图**：按日策略净值的月末值计算月收益，公式为 `month_end_equity(m) / month_end_equity(m-1) - 1`；首个不完整月使用 `month_end_equity / initial_cash - 1`。默认使用按月份排列的柱状图，正收益红色、负收益绿色、零收益灰色，0 轴清晰可见；悬停显示月份、月初/上月末净值、月末净值和月收益率。标题旁展示盈利月份数、亏损月份数及最佳/最差月份。手续费开关改变后必须从更新后的日净值重新聚合，不得对交易收益简单按月求和。
5. **退出原因计数图**：分别统计止盈、止损和反向死叉退出次数；无已平仓交易时展示清晰空状态。
6. **各股票策略夏普比率对比图**：横向条形图按夏普比率从高到低排序；0 轴清晰可见，正值与负值使用不同颜色，每根条显示股票简称、代码和两位小数值。

前四张图是用户指定的核心回测图，必须同时出现在 Notebook 和网页应用中，且由同一份逐日回测结果生成。

P0 顶部必须展示七项绩效指标。为保持信息层级，第一行放四张核心风险收益卡，第二行放三张交易质量/基准比较卡：

| 卡片 | 主值 | 辅助信息 |
|---|---|---|
| 总收益率 | 百分比，保留 2 位 | 期末净值、初始资金 |
| 年化收益率 | 百分比，保留 2 位 | 有效交易日数；不足一年标注“短样本年化” |
| 最大回撤 | 负百分比，保留 2 位 | 回撤峰值日与谷底日 |
| 夏普比率 | 数值，保留 2 位或 `N/A` | `Rf=0`、252 日年化 |
| 胜率 | 百分比，保留 1–2 位或 `N/A` | 盈利笔数 / 已平仓总笔数 |
| 盈亏比 | 倍数，保留 2 位或 `N/A` | 平均盈利 / 平均亏损绝对值 |
| 超额收益 | 百分比，保留 2 位 | 同区间策略收益 − 买入持有收益 |

卡片颜色不能单独表达好坏；负收益、负夏普和较大回撤还需有符号或文字。七项数值都来自当前选中股票，不得与跨股票对比图的共同区间结果混用；若主看板区间与对比区间不同，应在夏普卡旁提供“当前股票所选区间”，在对比图副标题提供“共同区间”。每项指标名称旁提供可点击的信息按钮，移动端也能打开对应说明。

交互要求：四张时间序列图共用一致的日期范围；价格图、净值图和回撤图的悬停日期尽可能联动。图例可区分三种退出原因；参数变化后保持可用的当前缩放范围；窄屏下单列布局，不出现横向溢出。A 股涨跌颜色采用红涨绿跌，但买入/卖出符号还应通过形状和文字区分，不能只依赖颜色。

交易表按退出日期倒序，支持按退出原因筛选；期末浮动持仓放在表格上方单独提示，不伪装成已平仓交易。

### 8.4 状态与错误提示

- `x >= y`：提示“快线周期必须小于慢线周期”，保留上一组有效结果。
- x 或 y 为空、为小数或非数字：提示“SMA 周期必须为整数”，保留上一组有效结果。
- `y > N-2`：提示“当前共有 N 个交易日；慢线最多为 N-2，才能保留交叉判断和下一日成交”，不得静默截断。
- `start_date > end_date`：提示“起始日期不能晚于结束日期”，保留上一组有效结果。
- 日期格式无效或裁剪后区间没有交易日：阻止重算并给出具体原因。
- 实际区间只有 1 个交易日：提示“至少需要 2 个交易日计算收益”；图表可展示数据，但年化收益、波动率和夏普显示 `N/A`。
- RSI 周期必须为 2–50 的整数，RSI 上限必须满足 `1 <= threshold <= 99`。
- 止盈必须在 0.1%–100%，止损必须在 0.1%–99%；关闭的规则不阻止其他参数重算。
- 初始资金必须为有限正数且在规定范围内。
- 三种退出规则均关闭时显示醒目但非阻断警告；交易记录必须清楚标示期末仍持仓。
- 没有金叉、没有成交或没有已平仓交易：显示空状态，不报错。
- 某月没有任何价格变化或交易时，月收益显示 0%；数据范围之外的月份不补零柱。
- 数据加载失败：展示可操作提示，包括使用本地静态服务器启动的方法。
- 跨股票对比数据部分缺失：其余股票正常绘制，缺失股票列在图下并说明原因；不得让整个主看板失败。
- 正在计算时只对结果区显示轻量状态；241 行数据下目标响应时间 `<100ms`（不含首次资源加载）。

## 10. 测试计划

### 10.1 单元与边界测试

- SMA 首个有效位置和窗口值正确。
- Wilder RSI 初始化、全涨、全跌、横盘及 `RSI == 70` 边界正确。
- 金叉/死叉包含相等值过渡，且不会连续重复触发。
- 金叉但 RSI 无效、等于 70 或高于 70 时不买入。
- 信号在 `t` 收盘生成，只能在 `t+1` 开盘成交。
- 单独触发止盈、单独触发止损、跳空越过阈值均使用规定成交价。
- 同日止盈止损双触发时止损优先并写入歧义标记。
- 反向信号排队至下一开盘，且开盘卖出优先于日内检查。
- 万二手续费开启时双边费用为成交额 `0.02%`，关闭时为 0；切换不改变信号和成交价，只改变可买股数、费用、净值和绩效。
- 自定义止盈止损值正确转换为小数并生成对应触发价；关闭开关后该原因不会出现在交易记录中。
- RSI 周期、上限和过滤开关分别影响入场；阈值相等时不入场。
- 反向死叉关闭时不生成对应卖出订单；三个退出规则全关时只做期末估值。
- 最后一日信号不成交；期末持仓只估值、不计入已平仓胜率。
- 零交易、全赢、全亏、零波动等指标不会产生异常或无穷值。
- 参数 `x >= y`、负数、非整数和超出样本窗口时被拒绝。
- 对多组非预设值（如 `7/23`、`13/47`）运行成功，证明输入未被限制在快捷组合。
- 起始日前数据只用于 SMA/RSI 预热，不能产生持仓、交易、净值或费用。
- 非交易日起止日期分别正确顺延/回退，且界面显示实际采用日期。
- 结束日后的价格和信号不影响任何回测结果；结束日待成交信号被正确标记。

- 七项指标均有用户可访问的介绍文案，阈值说明包含短样本和非保证提示。
- 策略/基准净值、价格均线信号、回撤和月度收益四张核心图均可见，且与逐日结果和交易明细一致。
- 月度收益使用月末策略净值计算；逐月复利结果应与总收益率在浮点容差内一致。
- 夏普比率对比图使用同参数、同共同区间；逐只抽查图中值与独立回测结果一致。
- 调整 `x/y` 后四张核心卡和对比图同时更新，且 `N/A` 股票不会被错误排序为 0。
- 切换万二手续费后无需刷新页面，策略与基准均按新费用口径重算；开启与关闭结果可由独立计算复核。
- 页面明确显示数据范围、前复权（收益口径）、成交假设、费用假设和免责声明。
- 页面在常见桌面宽度和 375px 移动端宽度下可阅读。
